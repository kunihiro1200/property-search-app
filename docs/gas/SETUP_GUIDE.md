# Google Apps Script セットアップガイド

## 概要

物件リストと業務リストのスプレッドシートに変更があったとき、自動的にDBへ同期します。

- **物件リスト**: セル編集・行追加 → `/api/property-listing-sync/manual`
- **業務リスト**: セル編集・行追加 → `/api/work-tasks/sync`

---

## 1. 物件リストスプレッドシートへの設置

### 手順

1. [物件リストスプレッドシート](https://docs.google.com/spreadsheets/d/1tI_iXaiLuWBggs5y0RH7qzkbHs9wnLLdRekAmjkhcLY) を開く

2. メニュー → **拡張機能** → **Apps Script** をクリック

3. エディタが開いたら、`property-listing-trigger.gs` の内容を全てコピーして貼り付ける

4. 保存（Ctrl+S）

5. **トリガーを設定する**:
   - 左メニューの時計アイコン（トリガー）をクリック
   - 「トリガーを追加」ボタンをクリック
   - **トリガー1（編集時）**:
     - 実行する関数: `onPropertyEdit`
     - イベントのソース: **スプレッドシートから**
     - イベントの種類: **編集時**
     - 保存
   - 「トリガーを追加」ボタンをクリック
   - **トリガー2（変更時）**:
     - 実行する関数: `onPropertyChange`
     - イベントのソース: **スプレッドシートから**
     - イベントの種類: **変更時**
     - 保存

6. 権限の承認を求められたら「許可」をクリック

---

## 2. 業務リストスプレッドシートへの設置

### 手順

1. [業務リストスプレッドシート](https://docs.google.com/spreadsheets/d/1MO2vs0mDUFCgM-rjXXPRIy3pKKdfIFvUDwacM-2174g) を開く

2. メニュー → **拡張機能** → **Apps Script** をクリック

3. `work-task-trigger.gs` の内容を全てコピーして貼り付ける

4. **物件番号の列番号を確認・修正**:
   ```javascript
   var PROPERTY_NUMBER_COLUMN = 1; // A列=1、B列=2、...
   ```
   業務リストスプレッドシートの物件番号が何列目にあるか確認して変更する

5. 保存（Ctrl+S）

6. **トリガーを設定する**（物件リストと同様）:
   - `onWorkTaskEdit` → 編集時
   - `onWorkTaskChange` → 変更時

---

## 3. 動作確認

### 物件リストのテスト

GASエディタで `testFullSync` 関数を選択して実行ボタンをクリック。
ログに `✅ 物件リスト同期成功` と表示されれば成功。

### 業務リストのテスト

GASエディタで `testFullSync` 関数を選択して実行ボタンをクリック。
ログに `✅ 業務リスト全件同期成功` と表示されれば成功。

---

## 注意事項

- **デバウンス**: 同じセルを連続編集しても5秒以内は1回しか同期しない（API負荷対策）
- **新規行追加**: `onChange` トリガーの `INSERT_ROW` イベントで検知する。一番下に追加された行も確実に検知される
- **ヘッダー行**: 1行目の編集は無視される
- **他のシート**: 「物件」「業務依頼」シート以外の編集は無視される
