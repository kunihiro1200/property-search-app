# 業務リスト日付パース修正 - 要件定義

## 1. 概要

業務リスト（work_tasks）の同期において、スプレッドシートの日付フィールドが正しくパースされず、データベースに空文字列として保存される問題を修正します。

## 2. 背景

### 現在の問題

AA12495において以下のフィールドがスプレッドシートと同期されていません：

1. **サイト登録締め日**: スプレッドシート `"2026/02/15"` → データベース `""`（空）
2. **サイト登録納期予定日**: スプレッドシート `"2026/02/15 18:05:00"` → データベース `""`（空）

### 根本原因

`WorkTaskColumnMapper.ts`の`parseDate`メソッドが、以下の形式に対応していません：

- `YYYY/MM/DD HH:MM:SS`形式（例: `"2026/02/15 18:05:00"`）

現在の正規表現 `/^\d{4}\/\d{1,2}\/\d{1,2}$/` は、時刻部分がある場合にマッチしません。

## 3. 対象システム

**システム**: 業務リスト（Work Task Management）

**影響範囲**:
- ✅ 業務リスト専用のファイルのみを変更
- ❌ 他のシステム（売主リスト、物件リスト、買主リスト、物件公開サイト）には影響を与えない

**対象ファイル**:
- `backend/src/services/WorkTaskColumnMapper.ts` - 日付パース処理

## 4. ユーザーストーリー

### US-1: 時刻付き日付のパース

**As a** システム管理者  
**I want** スプレッドシートの時刻付き日付（`YYYY/MM/DD HH:MM:SS`）を正しくパースしたい  
**So that** 業務リストのデータが正確にデータベースに同期される

**受け入れ基準**:
1. `"2026/02/15 18:05:00"`が`"2026-02-15"`としてパースされる
2. `"2026/02/15"`が`"2026-02-15"`としてパースされる（既存の動作を維持）
3. 時刻部分は無視される（日付のみを保存）

### US-2: 既存の日付形式のサポート維持

**As a** システム管理者  
**I want** 既存の日付形式のサポートを維持したい  
**So that** 既存のデータが正しく同期され続ける

**受け入れ基準**:
1. `YYYY-MM-DD`形式が正しくパースされる
2. `YYYY/MM/DD`形式が正しくパースされる
3. `MM/DD`形式が正しくパースされる（現在の年を使用）
4. その他の形式は`Date`オブジェクトでパースされる

### US-3: AA12495の同期修正

**As a** システム管理者  
**I want** AA12495のデータを再同期したい  
**So that** スプレッドシートとデータベースのデータが一致する

**受け入れ基準**:
1. 修正後、AA12495を再同期する
2. `site_registration_deadline`が`"2026-02-15"`として保存される
3. `site_registration_due_date`が`"2026-02-15"`として保存される
4. 他のフィールドも正しく同期される

- 他のシステム（売主リスト、物件リスト、買主リスト、物件公開サイト）には影響を与えない

### NFR-2: 後方互換性

- 既存の日付形式のサポートを維持する
- 既存のデータが正しく同期され続ける

### NFR-3: パフォーマンス

- 日付パース処理のパフォーマンスを維持する
- 正規表現のパフォーマンスを最適化する

## 6. 制約事項

### 技術的制約

1. **日付のみを保存**: 時刻部分は無視し、日付のみを`YYYY-MM-DD`形式で保存する
2. **既存の動作を維持**: 既存の日付形式のサポートを維持する
3. **システム隔離**: 業務リスト専用のファイルのみを変更する

### ビジネス制約

1. **データの整合性**: スプレッドシートとデータベースのデータが一致する必要がある
2. **既存データの保護**: 既存のデータが正しく同期され続ける必要がある

## 7. 対象外

以下は本要件の対象外です：

1. 時刻情報の保存（日付のみを保存）
2. タイムゾーンの処理（日本時間を前提）
3. 他のシステムの日付パース処理の変更
