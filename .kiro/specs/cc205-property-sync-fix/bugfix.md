# バグ修正要件ドキュメント

## はじめに

公開物件サイト（property-site-frontend）において、新規追加された物件CC205が表示されない問題を修正する。
スプレッドシートに追加された物件がデータベースに同期されていないか、同期されていてもサイトに表示されない状態になっている可能性がある。

本バグは、公開物件サイトのバックエンド（`backend/api/`）が対象であり、売主管理システム（`backend/src/`）は対象外とする。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN スプレッドシートに物件CC205が追加されている THEN システムは公開物件サイトの一覧にCC205を表示しない

1.2 WHEN ユーザーが公開物件サイト（`/public/properties`）にアクセスする THEN システムはCC205を含まない物件一覧を返す

1.3 WHEN `property_listings`テーブルにCC205が存在しない THEN システムはCC205をデータベースに同期していない状態になっている

1.4 WHEN `property_listings`テーブルにCC205が存在するが`is_hidden = true`になっている THEN システムはCC205を公開物件一覧から除外している

### 期待される動作（正しい動作）

2.1 WHEN スプレッドシートに物件CC205が追加されている THEN システムは`property_listings`テーブルにCC205を同期するべきである

2.2 WHEN `property_listings`テーブルにCC205が存在し`is_hidden = false`である THEN システムは公開物件サイトの一覧にCC205を表示するべきである

2.3 WHEN CC205の`atbb_status`が「公開中」を含む THEN システムはCC205を公開物件として表示するべきである

2.4 WHEN CC205の同期が完了している THEN システムは`/api/public/properties`エンドポイントでCC205を返すべきである

### 変更されない動作（リグレッション防止）

3.1 WHEN 既存の公開物件（CC205以外）が存在する THEN システムは引き続きそれらの物件を正常に表示するべきである

3.2 WHEN `is_hidden = true`の物件が存在する THEN システムは引き続きそれらの物件を公開一覧から除外するべきである

3.3 WHEN 売主管理システム（`backend/src/`）が動作している THEN システムはその動作に影響を与えないべきである

3.4 WHEN 既存の同期ロジック（`EnhancedAutoSyncService`）が動作している THEN システムはその動作を変更しないべきである

3.5 WHEN 公開物件サイトの他のAPIエンドポイント（画像取得、詳細取得など）が動作している THEN システムはそれらの動作を変更しないべきである
