# プロジェクト名変更時のキャッシュクリアルール（絶対に守るべきルール）

## ⚠️ 最重要：プロジェクト名を変更したらキャッシュをクリアする

**絶対に守るべき原則**:
- **`package.json`のプロジェクト名を変更した場合、必ずキャッシュをクリアする**
- **キャッシュをクリアしないと、古いバージョンのコードが表示され続ける**
- **ブラウザとViteの両方のキャッシュをクリアする必要がある**

---

## 📋 プロジェクト名変更時の必須手順

### ステップ1: 開発サーバーを停止

```bash
# 開発サーバーを停止（Ctrl + C）
```

### ステップ2: Viteキャッシュをクリア

```bash
cd frontend
rm -rf .vite
# または
Remove-Item -Recurse -Force .vite
```

### ステップ3: node_modulesを再インストール（必要な場合）

```bash
cd frontend
rm -rf node_modules
npm install
```

### ステップ4: 開発サーバーを再起動

```bash
cd frontend
npm run dev
```

### ステップ5: ブラウザのキャッシュをクリア

1. **完全にキャッシュをクリア**: Ctrl + Shift + Delete → すべてのキャッシュをクリア
2. **ブラウザを再起動**
3. ページを開く

---

## 🚨 プロジェクト名変更の影響

### 影響を受けるもの
1. **Viteのキャッシュ**: `.vite`ディレクトリ
2. **ブラウザのキャッシュ**: ローカルストレージ、セッションストレージ、キャッシュ
3. **node_modules**: 依存関係のキャッシュ

### 症状
- コードを変更しても反映されない
- 古いバージョンのコードが表示される
- 新しい機能が表示されない
- エラー「vite is not recognized」が表示される

---

## 📝 過去の問題例

### 2026年2月14日: プロジェクト名を`admin-frontend`に変更後、予約値下げ完了ボタンが表示されない

**原因**:
- `package.json`のプロジェクト名を`seller-list-management-frontend`から`admin-frontend`に変更
- Viteとブラウザのキャッシュが古いバージョンを参照していた
- `node_modules`が正しくインストールされていなかった

**症状**:
- 予約値下げ完了ボタンが表示されない
- コードを変更しても反映されない
- エラー「vite is not recognized」が表示される

**解決方法**:
1. 開発サーバーを停止
2. `.vite`ディレクトリを削除
3. `node_modules`を削除して再インストール
4. 開発サーバーを再起動
5. ブラウザのキャッシュをクリア

2. `.vite`ディレクトリを削除
3. 開発サーバーを再起動
4. ブラウザのキャッシュをクリア（Ctrl + Shift + Delete）
5. ブラウザを再起動

### 問題3: 新しい機能が表示されない

**原因**: ブラウザのキャッシュが古い

**解決方法**:

## 📊 キャッシュクリアの優先順位

| 問題 | 優先順位 | 対応 |
|------|---------|------|
| エラー「vite is not recognized」 | 最高 | node_modulesを再インストール |
| コードを変更しても反映されない | 高 | .viteディレクトリを削除 |
| 新しい機能が表示されない | 中 | ブラウザのキャッシュをクリア |

---

## まとめ

**絶対に守るべきルール**:

1. **`package.json`のプロジェクト名を変更した場合、必ずキャッシュをクリアする**
2. **開発サーバーを停止してから`.vite`ディレクトリを削除する**
3. **エラー「vite is not recognized」が表示された場合は、`node_modules`を再インストールする**
4. **ブラウザのキャッシュをクリアして、ブラウザを再起動する**
5. **変更が反映されているか必ず確認する**

**このルールを徹底することで、プロジェクト名変更後もスムーズに開発を続けられます。**
